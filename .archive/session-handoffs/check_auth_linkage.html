🔍 Check Auth Linkage Status

Diagnose why Build 9 isn't working

What this checks:
  • Do auth.users exist for demo accounts?
  • Are patients/therapists tables linked to auth.users?
  • Is patient assigned to therapist?

Step 1: Check Current Status

-- Check Auth Linkage Status -- 1. Check if auth users exist SELECT 'Auth Users' as check_type, COUNT(*) as count, array_agg(email) as emails FROM auth.users WHERE email IN ('demo-athlete@ptperformance.app', 'demo-pt@ptperformance.app'); -- 2. Check patient linkage SELECT 'Patient Linkage' as check_type, p.id as patient_id, p.email, p.user_id as patient_user_id, p.therapist_id, au.id as auth_user_id, CASE WHEN p.user_id IS NULL THEN '❌ NOT LINKED' WHEN p.user_id = au.id THEN '✅ LINKED' ELSE '⚠️ MISMATCH' END as status FROM patients p LEFT JOIN auth.users au ON au.email = p.email WHERE p.email = 'demo-athlete@ptperformance.app'; -- 3. Check therapist linkage SELECT 'Therapist Linkage' as check_type, t.id as therapist_id, t.email, t.user_id as therapist_user_id, au.id as auth_user_id, CASE WHEN t.user_id IS NULL THEN '❌ NOT LINKED' WHEN t.user_id = au.id THEN '✅ LINKED' ELSE '⚠️ MISMATCH' END as status FROM therapists t LEFT JOIN auth.users au ON au.email = t.email WHERE t.email = 'demo-pt@ptperformance.app'; -- 4. Check if patient has therapist assigned SELECT 'Patient-Therapist Link' as check_type, p.email as patient_email, t.email as therapist_email, CASE WHEN p.therapist_id IS NULL THEN '❌ NO THERAPIST' WHEN p.therapist_id = t.id THEN '✅ ASSIGNED' ELSE '⚠️ WRONG THERAPIST' END as status FROM patients p LEFT JOIN therapists t ON t.email = 'demo-pt@ptperformance.app' WHERE p.email = 'demo-athlete@ptperformance.app';

Step 2: If Any ❌ Found, Run This Fix

-- Fix Auth Linkages -- Update patient user_id to match auth.users UPDATE patients p SET user_id = au.id FROM auth.users au WHERE p.email = au.email AND p.email = 'demo-athlete@ptperformance.app'; -- Update therapist user_id to match auth.users UPDATE therapists t SET user_id = au.id FROM auth.users au WHERE t.email = au.email AND t.email = 'demo-pt@ptperformance.app'; -- Link patient to therapist UPDATE patients p SET therapist_id = t.id FROM therapists t WHERE p.email = 'demo-athlete@ptperformance.app' AND t.email = 'demo-pt@ptperformance.app'; -- Verify fixes SELECT 'VERIFICATION' as status, (SELECT COUNT(*) FROM patients WHERE email = 'demo-athlete@ptperformance.app' AND user_id IS NOT NULL) as patient_linked, (SELECT COUNT(*) FROM therapists WHERE email = 'demo-pt@ptperformance.app' AND user_id IS NOT NULL) as therapist_linked, (SELECT COUNT(*) FROM patients WHERE email = 'demo-athlete@ptperformance.app' AND therapist_id IS NOT NULL) as patient_has_therapist;
