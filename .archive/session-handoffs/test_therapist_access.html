🔍 Test Therapist RLS Access

Check if RLS policies allow therapist to see patients

What this checks:
  • RLS policies exist for patients table
  • Therapist can see their assigned patients
  • Patient-therapist linkage is correct
-- Test Therapist RLS Access -- 1. Check RLS policies exist SELECT tablename, policyname, cmd as command, qual as using_expression FROM pg_policies WHERE schemaname = 'public' AND tablename = 'patients' ORDER BY policyname; -- 2. Check therapist-patient linkage (bypassing RLS) SELECT 'Linkage Check' as test, t.email as therapist_email, t.user_id as therapist_user_id, p.email as patient_email, p.user_id as patient_user_id, p.therapist_id, CASE WHEN p.therapist_id = t.id THEN '✅ Correctly linked' ELSE '❌ NOT linked' END as link_status FROM therapists t CROSS JOIN patients p WHERE t.email = 'demo-pt@ptperformance.app' AND p.email = 'demo-athlete@ptperformance.app'; -- 3. Check if data exists (without RLS) SELECT 'Data Exists' as test, COUNT(*) as patient_count FROM patients p JOIN therapists t ON p.therapist_id = t.id WHERE t.email = 'demo-pt@ptperformance.app'; -- 4. Check programs for patient SELECT 'Programs Check' as test, pr.name, pr.status, p.email as patient_email FROM programs pr JOIN patients p ON pr.patient_id = p.id WHERE p.email = 'demo-athlete@ptperformance.app'; -- 5. Show all auth user IDs (to verify they exist) SELECT 'Auth Users' as type, id as user_id, email FROM auth.users WHERE email IN ('demo-pt@ptperformance.app', 'demo-athlete@ptperformance.app') ORDER BY email;
