# PT App – Key User Stories

These stories are used by agents to create issues and code.

---

## 1. Patient Stories

### P1 – See Today's Session
"As a patient, I want to open the app and immediately see what I must do today."

### P2 – Log Sets & Pain Quickly
"I want to log my reps and pain without digging through menus."

### P3 – Review My Progress
"I want simple charts showing my adherence and pain trend."

---

## 2. Therapist Stories

### T1 – Create a Multi-Phase Program
"As a PT, I can create phases → sessions → exercises."

### T2 – Review Patient Flags
"I'm alerted when pain spikes, adherence drops, or velocity declines."

### T3 – Approve Plan Changes
"When the agent proposes a change, I approve or reject from phone."

### T4 – Mobile Charting
"I can update notes and see patient logs on iPad."

---

## 3. System Stories (Agents)

### S1 – Keep Linear in Sync
"Agents check Linear before coding anything."

### S2 – Propose Only Structured Changes
"Agents create Plan Change Request issues, not free text."

### S3 – Execute Assigned Zones
"Agents perform only tasks within their zone."

### S4 – Update Linear
"When code or schema is updated, update Linear issue with status + comment."
