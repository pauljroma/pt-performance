## Architecture

### Flag Computation Flow

```
1. Client requests /patient-summary/:patientId or /pt-assistant/summary/:patientId
   ↓
2. Server calls getFlagSummary(patientId)
   ↓
3. getFlagSummary calls computeFlags(patientId)
   ↓
4. computeFlags fetches data from Supabase:
   - Patient details
   - Pain logs (last 10)
   - Bullpen logs (last 10)
   - Adherence data
   ↓
5. Evaluates each flag type:
   - evaluatePainFlags(painLogs)
   - evaluateVelocityFlags(bullpenLogs, position)
   - evaluateCommandFlags(bullpenLogs, position)
   - evaluateAdherenceFlags(adherencePct, patientId)
   - evaluateThrowingPainFlags(bullpenLogs, position)
   ↓
6. Returns sorted flags (HIGH → MEDIUM → LOW)
   ↓
7. Server checks for HIGH severity flags
   ↓
8. For each HIGH flag:
   - Calls createPlanChangeRequest(flag, patient)
   - Creates Linear issue with zone-4b label
   - Sets state to "In Review"
   - Logs PCR creation
   ↓
9. Returns response with flags and recommendations
```

### Flag Object Structure

```javascript
{
  flag_type: 'pain_high' | 'velocity_critical_drop' | 'throwing_pain_high' | ...,
  severity: 'HIGH' | 'MEDIUM' | 'LOW',
  rationale: 'Pain score 7/10 reported on 2025-03-15. Immediate review required.',
  triggered_at: '2025-03-15T15:30:00Z',
  metric_value: 7,
  metric_name: 'pain_score',
  patient_id: 'uuid',
  context: {  // Optional
    pitch_type: '4-FB',
    velocity: 87,
    pitch_count: 12
  }
}
```
